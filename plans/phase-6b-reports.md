# Phase 6B: Report Exports — Audit-Ready PDF & Excel Reports

> Parent: PLAN.md Phase 6B
> Status: COMPLETE

## Problem Statement

External auditors need downloadable, formatted reports to verify farm compliance — particularly around medication withdrawal periods, treatment traceability, and discarded milk accountability. The existing JSON export (`GET /api/export`) is a raw data dump for backups, not suitable for audit presentation.

## Solution

Build 7 report endpoints under `GET /api/reports/*` (admin-only) that return PDF or Excel files based on a `format` query param. Start with the 3 audit-critical reports, then add 4 more. A shared `reportService.js` provides PDF/Excel generation utilities. A new `ReportsView.vue` admin page lets the user pick a report type, date range, and format, then downloads the file.

**Tech:** `pdfkit` (PDF), `exceljs` (Excel) — server-side only, zero client bundle impact.

**No migrations needed** — all data already exists in current tables.

## Phase 6B.1: Foundation — Dependencies & Report Service Skeleton

**Goal:** Install deps, create shared report generation utilities, scaffold the route file.

### 6B.1.1 Install dependencies
- `npm install pdfkit exceljs` (server-only)

### 6B.1.2 Create `server/services/reportService.js`
Shared utilities:
- `getFarmName()` — reads `farm_name` from `app_settings` table
- `formatDate(dateStr)` / `formatDateTime(dateStr)` — display formatting
- `createPdfDocument({ title, farmName, dateRange, generatedBy })` — returns `{ doc, finalize }`. Sets up A4 landscape, farm name header, report title, date range line, "Generated by X on Y" line. `finalize()` adds page numbers and returns a `Buffer`.
- `drawPdfTable(doc, { columns, rows, summaryRow })` — renders a bordered table with header row, data rows (auto page-break), and optional bold summary row.
- `createExcelReport({ title, sheetName, farmName, dateRange, generatedBy, columns, rows, summaryRow })` — returns `Buffer`. Adds title rows (styled), header row (green bg, white text, frozen), data rows, optional summary row (bold + top border), auto column widths.

### 6B.1.3 Create `server/routes/reports.js` (skeleton)
- `router.use(authenticate)` + `router.use(authorize('admin'))`
- Shared Joi `querySchema`: `from` (required YYYY-MM-DD), `to` (required YYYY-MM-DD), `format` (pdf|xlsx, default pdf)
- `sendFile(res, buffer, filename, format)` helper — sets Content-Type + Content-Disposition

### 6B.1.4 Mount in `server/app.js`
- Add `app.use('/api/reports', require('./routes/reports'))` after the export line

## Phase 6B.2: Withdrawal Compliance Report

**Goal:** The most audit-critical report — proves milk was properly discarded during withdrawal periods.

### 6B.2.1 Data helper `getWithdrawalData(from, to)`
Query: `treatments` JOIN `cows` JOIN `users`, filtered to female cows with `withdrawal_end_milk` in range. Batch-fetch medications via `treatment_medications` JOIN `medications` (avoid N+1). For each treatment, count `milk_records` during withdrawal window — split into discarded vs not-discarded. Mark COMPLIANT or VIOLATION.

**Row fields:** Tag #, Cow Name, Treatment Date, Medication(s), Active Ingredient(s), Withdrawal End (Milk), Withdrawal End (Meat), Records During, Discarded Count, Violations, Status, Administered By

**Summary:** Total treatments, Compliant count, Violation count

### 6B.2.2 Endpoint `GET /api/reports/withdrawal-compliance`
Validate query → fetch data → generate PDF or Excel via reportService → sendFile

### 6B.2.3 Tests & Quality Gate
**Tests** (in `server/tests/reports.test.js`):
- 401 without token
- 403 for worker
- 400 when from/to missing
- 400 when format invalid
- PDF returns correct Content-Type + Content-Disposition
- XLSX returns correct Content-Type + Content-Disposition
- Empty report when no treatments in range
- COMPLIANT when all milk discarded during withdrawal
- VIOLATION when milk not discarded during withdrawal
- Includes medication names and active ingredients

**Quality gate:** lint, knip, refactor check (no N+1 queries, batch medication lookup), all tests pass

## Phase 6B.5: Remaining 4 Reports

**Goal:** Complete the full report suite — medication usage, milk production, breeding, herd health.

### 6B.5a Medication Usage Report
**Endpoint:** `GET /api/reports/medication-usage`
**Data:** Grouped by medication. Count treatments, total cost, % of total, avg cost per treatment.
**Columns:** Medication Name, Active Ingredient, Times Used, Total Cost, % of Treatments, Avg Cost/Treatment
**Summary:** Grand total treatments, Grand total cost

### 6B.5b Milk Production Report
**Endpoint:** `GET /api/reports/milk-production`
**Data:** Per-cow production summary. Group `milk_records` by `cow_id`.
**Columns:** Tag #, Cow Name, Total Litres, Days Recorded, Avg Daily Litres, Morning Avg, Afternoon Avg, Evening Avg
**Summary:** Herd total litres, Avg per cow, Total records

### 6B.5c Breeding & Reproduction Report
**Endpoint:** `GET /api/reports/breeding`
**Data:** All breeding events in range. Join `breeding_events` + `cows` + `users`.
**Columns:** Date, Tag #, Cow Name, Event Type, Sire/Semen ID, Inseminator, Cost, Preg Check Method, Notes
**Summary:** Total events, Breakdown by type

### 6B.5d Herd Health Summary Report
**Endpoint:** `GET /api/reports/herd-health`
**Data:** All health issues in range. Join `health_issues` + `cows` + `users`. Enrich issue type codes via `issue_types` table. Count linked treatments per issue.
**Columns:** Date Observed, Tag #, Cow Name, Issue Type(s), Severity, Status, Resolved Date, Days to Resolve, Reported By, Treatment Count
**Summary:** Total issues, By severity, Resolution rate, Avg days to resolve

### 6B.5.5 Tests & Quality Gate
**Tests:**
- Each of the 4 reports: PDF + XLSX correct headers (8 tests)
- medication-usage groups correctly
- milk-production computes per-cow + per-session averages
- breeding includes all event types
- herd-health enriches issue type codes to names + computes resolution time
- All 4 reject worker tokens

**Quality gate:** lint, knip, efficiency review (batch medication lookups, use `getIssueTypeDefMap()` pattern from analytics for health report), all tests pass
