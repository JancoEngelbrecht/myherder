What's required vs. optional

Backend (server/routes/animals.ts) currently requires only one field: tag_number. Everything else has a safe default or is nullable.

  • Required (1): tag_number
  • Needed to actually use the app (3): tag_number, sex (drives male/female UI gating), breed_type_id (drives breeding auto-dates)
  • Optional — add later when needed: name, date of birth, sire, dam, notes, purpose, life phase override, external flag

Minimum viable animal = tag + sex + breed. The batch create screen collects breed, sex, and status once as shared defaults, then only asks for tag numbers — everything else is filled in later by tapping an animal when the farmer encounters it.

Key interactions

  • Count stepper: farmer picks how many blocks to show (10, 20, 50…). The number can grow as they type (auto-add a new block when last one is filled).
  • Smart keyboard: tag inputs open numeric keypad on phone (inputmode="numeric").
  • Auto-number fallback: farmer without ear tags ticks the checkbox and gets RAM-001…RAM-020 auto-generated — no typing at all.
  • Duplicate detection: tags that already exist in the herd (or repeat within the batch) turn red and get skipped on submit, with a toast showing which ones.
  • Multi-batch in one session: after creating 8 rams, tap "Add another batch" and do 50 ewes. Same screen, different defaults.
  • Multi-delete: admin-only. Tap "Select" in list header → cards show checkboxes → pick mistakes → delete with confirm. Uses existing soft-delete (deleted_at) so errors are recoverable.
